[.underline]*Agendamento do Manual de Replicação de Log*

*Configurar e iniciar o modelo de trabalho*

. Copie o modelo de trabalho criado anteriormente.
+
.. Navegue até Recursos → Modelos.
.. Encontre o modelo de configuração ONTAP/CVO e, na extrema direita, clique em Copiar modelo
.. Clique em Editar modelo no modelo copiado e altere o nome para Manual de replicação de log.
.. Mantenha o mesmo inventário, projeto e credenciais para o modelo.
.. Selecione ora_replication_logs.yml como o manual a ser executado.
.. As variáveis permanecerão as mesmas, mas o IP do cluster CVO precisará ser definido na variável dst_cluster_ip.
.. Clique em Salvar.


. Agende o modelo de trabalho.
+
.. Navegue até Recursos → Modelos.
.. Clique no modelo Log Replication Playbook e depois clique em Schedules no conjunto superior de opções.
.. Clique em Adicionar, adicione o Nome Agendado para Replicação de Log, escolha a Data/hora de início no início da hora, escolha seu Fuso horário local e Frequência de execução.  A frequência de execução será sempre que a replicação do SnapMirror será atualizada.


+

NOTE: É recomendável definir o agendamento do log para atualizar a cada hora para garantir a recuperação para a última atualização horária.
